JavaScript Data Types

There are 8 JavaScript data types (you learned about 7 of them for homework). 7 primitive types, and one object type.

Primitive JavaScript Data Types

Primitive data types (primitive value, primitive data type) is data that has no object and no methods.

There are 7 primitive data types in JavaScript: string, boolean, number, bigInt, symbol, null, and undefined.

All primitive data types are immutable. They cannot be changed. The variable that hold the primitive value can be re-assigned another value, but the value itself cannot be changed in the ways that objects, functions, and arrays can be changed.

Primitive JavaScript Data Types: string

The string primitive data type, which you have already encountered, looks like so:

'Maria' or "Maria" or `Maria`.

string is used to represent text data. Each string is made up of a set of "elements" of 16-bit unsigned integer values (the indexes). That means that each element occupies a position in the String. The first element is at index 0, the second at index 1, etc. The length of the String is the number of elements in it.

JavaScript strings are immutable. Once created, they cannot be changed. BUT, it is possible to create another string based on an operation on the original string: a substring of the original by using String.substr() or concatenating strings with String.concat().

Primitive JavaScript Data Types: number

JavaScript has two built in number types: BigInt and Number.

Primitive JavaScript Data Types: Number

  1. The Number type is a double-precision 64-bit binary format IEEE 754 value (numbers between -(253 - 1) and 253 - 1)
  2. Number can represent floating-point numbers.
  3. The number type has 3 symbolic values: +Infinity, -Infinity, and NaN.
  4. The constants Number.MAX_VALUE or Number.MIN_VALUE check for the largest available value or smallest available value within Â±Infinity.
  5. The number type has only one integer with two representations: 0 is represented as both -0 and + 0. 0 is an alias for +0. This usually does not matter, as -0 === +0. The only time you might notice something different is when you divide by 0.

ECMAScript 2015 introduced Number.isSafeInteger(), Number.MAX_SAFE_INTEGER

These built-in methods check whether a number is in the double precision floating point range. Beyond that, integers are no longer safe and will only be a double precision floating point approximation of the actual value. and Number.MIN_SAFE_INTEGER.

Primitive JavaScript Data Types: Number (contd)

There is only one integer in the number type that is represented twice, and that is 0. There is -0 and +0. 0 really stands for +0.

Primitive JavaScript Data Types: BigInt

  • BigInt is a number type that can represent integers beyond the safe integer limit for Numbers.
  • BigInt is created by appending n to the end of an integer or by calling the constructor.
  • With the introduction of BigInts, you can operate beyond the Number.max_safe_integer (max_safe_integer should be in caps).
  • You can use +, *, -, **, and % with BigInts - just like with Numbers. A BigInt is not strictly equal to a Number, only loosely so.
  • A BigInt behaves like a number in cases where it is converted to Boolean with if, ||, &&, Boolean, and !.

Primitive JavaScript Data Types: symbol

Symbols are new to JavaScript since ECMAScript 2015. A Symbols is a unique and immutable primitive value and may be used as the key of an Object property. In some programming languages, Symbols are called "atoms".

Syntax:

Symbol(description)

Symbol(): is the Symbol constructor. The constructor method is a special method for creating and initializing an object created within a class. We will be getting into objects created with classes later in the course as well as implementing them!

description: optional. A string. It is the description of the symbol, which can be used for debugging, but not to access the symbol itself.

let sym1 = Symbol()

let sym2 = Symbol('foo')

let sym3 = Symbol('foo')

Primitive JavaScript Data Types: null

  • The null type as only one value: null.
  • The value null represents the absence of any object value.
  • it is treated as falsy for any boolean operations.
  • null is not written as an identifier for a property of the global object, like undefined can be.
  • null express a lack of identification and absence of value. This means and indicates that a variable is not pointing to any object whatsover.
  • In APIs, null is found where an object is expected but not there.

What Does falsy Mean?

  • A falsy value is a value that is considered false when encountered in a Boolean context.
  • And referring to the previous slide, JavaScript uses Type Conversion behind the scenes to coerce any value to a Boolean in contexts that require it, such as conditionals and loops.

falsy Values

There are 7 falsy values in JavaScript:

  • false: the keyword false
  • 0: the number 0
  • 0n: BigInt, when used as a boolean, follows the same rule as a Number. 0n is falsy.
  • "", '', ``: This is an empty string (three different representationsas discussed in the string primitive). The length of the string is 0, which means that it indeed is empty.
  • null: the absence of any value, as discussed in slide on mull.
  • undefined: refers to when something is undefined, and refers to the primitve value.
  • NaN: not a number.

JavaScript Data Types: object

In computer science, an object is a value in memory which could be referenced by an identifier.

Ask the students to explain to me what the identifier is.

JavaScript Data Types: object (contd)

  • In JavaScript, objects can be regarded as a collection of properties.
  • With the object literal syntax, a limited set of properties can be intialized, then later removed or added.
  • Property values can be of any type or other objects. This means being able to build complex data structures, unlike with primitives such as strings.
  • Properties are defined using key values. A key value is either a String or a Symbol value.
  • Object properties (aka keys) can be any string, including an empty one.
  • Property names which are NOT valid identifiers or numbers cannot be accessed with a dot (.) property, but can be set with the array like notation [''] and encapsulated by single or double quotes.

JavaScript Data Types: types of object properties

There are two types of object properties which have certain attributes; the data property and the accessor property.

JavaScript Data Types: object Data Property

Associates a key with a value. To view all the attributes associated with the Data property of an object, please visit Data property. The actual Attribute(s) are actually used by the JavaScript V8 Engine, so developers can't access Attribute. However, they can use the Object.defineProperty() built in method, which is really cool.

JavaScript Data Types: object: Object.defineProperty()

The (static) method Object.defineProperty() defines a new object property directly on an object OR modifies an existing property on an object and returns the object. This method is called directly on the Object constructor and not on an instance of type Object. We'll get into that more later when we get deeper into the object type.

JavaScript Data Types: object: Object.defineProperty() Syntax

Syntax:

Parameters:

descriptor: the descriptor for the property being defined or modified

Return value:

The object that was passed to the function

JavaScript Data Structures: regular objects, functions and dates

  • A JavaScript object is a mapping between keys and values. Keys are strings or Symbols, and values can be anything.
  • Functions are objects that are also callable.
  • The Date object: when representing dates, it is best to use the built in JavaScript Date object: const date = new Date()

JavaScript Data Structures: Indexed Collections: Arrays and Typed Arrays

  • Arrays are regular objects in which there is a relationship between integer-keyed properties and the length property. Remember that with Strings there is just the length property!
  • Arrays inherit from Array.proptotype, which means they have access to a bunch of convenient built in methods to manipulate arrays. Examples are indexOf(), searching a value in the array, push(), adding an element to the array. This makes Arrays great for representing lists or sets.
  • Typed Arrays are new to JavaScript in ECMAScript 2015, and are array-like objects that provide a mechanism for reading and writing raw binary data in memory buffers.

JavaScript Data Structures: Typed Arrays (contd)

  • Array objects grow and become smaller dynamically and can have any JavaScript value. JavaScript engines (i.e., V8 Engine) perform optimizations so that these arrays are fast.
  • However, when web applications become bigger and bigger and more and more powerful, adding things like video and audio manipulations, access to raw data using WebSockets(i.e. for live chat), it became evident that it would be great to make JavaScript able to quickly and easily manipulate raw binary data. This where typed arrays come in.
  • Each element in a typed array is a raw binary value in one of several supported formats. To learn more about typed arrays, please visit the Typed Arrays page on MDN.
