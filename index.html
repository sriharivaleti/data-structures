JavaScript Data Types

There are 8 JavaScript data types. 7 primitive types, and one object type. If something in JavaScript is not a primitive data type, it is an object!

Primitive JavaScript Data Types

Primitive data types (primitive value, primitive data type) is data that has no object and no methods.

There are 7 primitive data types in JavaScript: string, boolean, number, bigInt, symbol, null, and undefined.

All primitive data types are immutable. They cannot be changed. The variable that holds the primitive value can be re-assigned another value, but the value itself cannot be changed in the ways that objects, functions, and arrays can be changed.

Primitive JavaScript Data Types: string

The string primitive data type looks like so:

'Maria' or "Maria" or `Maria`.

String is used to represent text data. Each string is made up of a set of "elements" of 16-bit unsigned integer values (the indexes). That means that each element occupies a position in the String. The first element is at index 0, the second at index 1, etc. The length of the String is the number of elements in it.

JavaScript strings are immutable. Once created, they cannot be changed. BUT, it is possible to create another string based on an operation on the original string: a substring of the original by using String.substr() or concatenating strings with String.concat().

Primitive JavaScript Data Types: Number

JavaScript has two built in number types: BigInt and Number.

Primitive JavaScript Data Types: Number

  • The Number type is a double-precision 64-bit binary format IEEE 754 value (numbers between -(253 - 1) and 253 - 1)
  • Number can represent floating-point numbers.
  • The number type has 3 symbolic values: +Infinity, -Infinity, and NaN.
  • The constants Number.MAX_VALUE or Number.MIN_VALUE check for the largest available value or smallest available value within ±Infinity.
  • The number type has only one integer with two representations: 0 is represented as both -0 and + 0. 0 is an alias for +0. This usually does not matter, as -0 === +0. The only time you might notice something different is when you divide by 0. 

259 / -0
259 / +0

Primitive JavaScript Data Types: Number (contd)

ECMAScript 2015 introduced Number.isSafeInteger(), Number.MAX_SAFE_INTEGER, and Number.MIN_SAFE_INTEGER.

These built-in methods check whether a number is in the double precision floating point range. Beyond that, integers are no longer safe and will only be a double precision floating point approximation of the actual value.

Number: Floating Point Precision

Unlike other programming languages, JavaScript does not define different types of numbers like integers, short, long, floating point, etc. JavaScript numbers are always double precision floating point numbers.

Integers, numbers without a period or exponent notation, are accurate up to 15 digits.


const maxInt = 999999999999999;
const beyondMaxInt = 9999999999999999;

Beyond that, inaccuracy kicks in. And when it comes to decimal points, the max number is 17 before things get imprecise, but that is not always reliable.

Primitive JavaScript Data Types: BigInt

  • BigInt is a number type that can represent integers beyond the safe integer limit for Numbers.
  • BigInt is created by appending n to the end of an integer or by calling the constructor.
  • With the introduction of BigInts, you can operate beyond the Number.MAX_SAFE_INTEGER.
  • You can use +, *, -, **, and % with BigInts - just like with Numbers. A BigInt is not strictly equal to a Number, only loosely so.
  • A BigInt behaves like a number in cases where it is converted to Boolean with if, ||, &&, Boolean, and !.

Primitive JavaScript Data Types: symbol

Symbols are new to JavaScript since ECMAScript 2015. A Symbol is a unique and immutable primitive value and may be used as the key of an Object property. In some programming languages, Symbols are called "atoms".

Syntax:

Symbol(description)

Symbol(): is the Symbol constructor. The constructor method is a special method for creating and initializing an object created within a class.

description: optional. A string. It is the description of the symbol, which can be used for debugging, but not to access the symbol itself.

let sym1 = Symbol()

let sym2 = Symbol('foo')

let sym3 = Symbol('foo')

To learn more about Symbols, please visit the Symbol page on MDN.

Primitive JavaScript Data Types: null

  • The null type has only one value: null.
  • The value null represents the absence of any object value.
  • It is treated as falsy for any boolean operations.
  • null is not written as an identifier for a property of the global object, like undefined can be.
  • null expresses a lack of identification and absence of value. This means and indicates that a variable is not pointing to any object whatsoever.
  • In APIs, null is found where an object is expected but not there.

What Does falsy Mean?

  • A falsy value is a value that is considered false when encountered in a Boolean context.
  • JavaScript uses Type Conversion behind the scenes to coerce any value to a Boolean in contexts that require it, such as conditionals and loops.

falsy Values

There are 7 falsy values in JavaScript:

  • false: the keyword false
  • 0: the number 0
  • 0n: BigInt, when used as a boolean, follows the same rule as a Number. 0n is falsy.
  • "", '', ``: This is an empty string (three different representations as discussed in the string primitive). The length of the string is 0, which means that it indeed is empty.
  • null: the absence of any value, as discussed in slide on null.
  • undefined: refers to when something is undefined, and refers to the primitive value.
  • NaN: not a number.

The JavaScript global object

  • In JavaScript, there is always a global object defined and present. In a web browser, when scripts create global objects, they become a part of the Global Object. (In Node.js, this is not the case.)
  • The Window object is the Global Object in a web browser. Any global variables or functions can be accessed as properties of the Window object. As I mentioned earlier, we'll get more into depth about the Window object a bit later in the course.

globalThis

globalThis: is how you access the actual Window object in the browser instead of simply a reference to it.

It has not always been this way. You used to be able to access the Window object by typing Window or window in the browser. But now, if you go into Google Chrome, for example, and type Window or window and hit return, you will only get a reference to it.

In order to access the global Window object in the browser, you type:


globalThis

The reason for this is that historically, accessing the Global Object required different syntax on different browsers and environments, which then resulted in cross-browser compatibility issues. Now, we use globalThis to access the Window Object (and window is the child of Window).

JavaScript Data Types: object

In computer science, an object is a value in memory which could be referenced by an identifier.

JavaScript Data Types: object (contd)

  • In JavaScript, objects can be regarded as a collection of properties.
  • With the object literal syntax, a limited set of properties can be initialized, then later removed or added.
  • Property values can be of any type or other objects. This means being able to build complex data structures, unlike with primitives such as strings.
  • Properties are defined using key values. A key value is either a String or a Symbol value. In this course, we will be dealing with the key String.
  • Object properties (aka keys) can be any string, including an empty one.
  • Property names which are NOT valid identifiers or numbers cannot be accessed with a dot (.) property, but can be set with the array like notation [''] and encapsulated by single or double quotes. However, you don't really want to do that. Done this way, the code is hard to read/follow. You want clear keys for easier referencing.

JavaScript Data Types: object: Object.defineProperty()

The (static) method Object.defineProperty() defines a new object property directly on an object OR modifies an existing property on an object and returns the object. This method is called directly on the Object constructor and NOT on an instance of type Object.

JavaScript Data Types: object: Object.defineProperty() Syntax

Syntax:

Object.defineProperty(obj, prop, descriptor)

Parameters:

obj: the object on which to define the property

prop: the name or Symbol of the property to be defined

descriptor: the descriptor for the property being defined or modified

Return value:

The object that was passed to the function

JavaScript Data Types: object: Accessor Property

Associates a key with one of two accessor functions, get and set, to retrieve or store a value.

Again, the actual Attribute(s) is used by the JavaScript V8 Engine, so developers can't access Attribute. However, they can use the Object.defineProperty() built in method, which is really cool.

object Accessor Property: Object.defineProperty() Example


function ArchiveTemp() {
    let temp = null;
    let tempArchive = [];

    Object.defineProperty(this, 'temp', {
        get() {
            console.log(`get the temp!`);
            return temp;
        },
        set(value) {
            temp = value;
            tempArchive.push({val: temp});
        }
    });
    this.getArchiveTemp = function() {return tempArchive};
}

let arc = new ArchiveTemp();
console.log(arc.temp);
arc.temp = 52;
arc.temp = 23;

arc.getArchiveTemp();

JavaScript Data Structures: regular objects, functions and dates

  • A JavaScript object is a mapping between keys and values. Keys are strings or Symbols, and values can be anything.
  • Functions are objects that are also callable.
  • The Date object: when representing dates, it is best to use the built in JavaScript Date object: const date = new Date()

JavaScript Data Structures: Indexed Collections: Arrays and Typed Arrays

  • Arrays are regular objects in which there is a relationship between integer-keyed properties and the length property. Remember that with Strings there is just the length property!
  • Arrays inherit from Array.prototype, which means they have access to a bunch of convenient built in methods to manipulate arrays. Examples are indexOf(), searching a value in the array, push(), adding an element to the array. This makes Arrays great for representing lists or sets. We will get more into detail when we work with arrays.
  • Typed Arrays are new to JavaScript in ECMAScript 2015, and are array-like objects that provide a mechanism for reading and writing raw binary data in memory buffers. We won't be dealing with Typed Arrays in this course.

JavaScript Data Structures: Typed Arrays (contd)

  • Array objects grow and become smaller dynamically and can have any JavaScript value. JavaScript engines (i.e., V8 Engine) perform optimizations so that these arrays are fast.
  • However, when web applications become bigger and bigger and more and more powerful, adding things like video and audio manipulations, access to raw data using WebSockets(i.e. for live chat), it became evident that it would be great to make JavaScript able to quickly and easily manipulate raw binary data. This where typed arrays come in.
  • Each element in a typed array is a raw binary value in one of several supported formats. To learn more about typed arrays, please visit the Typed Arrays page on MDN.
